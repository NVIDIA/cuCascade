   ```cpp
   uint64_t data_batch::get_batch_id() const { return _batch_id; }

   bool data_batch::subscribe()
   {
     _subscriber_count.fetch_add(1, std::memory_order_relaxed);
     return true;
   }

   void data_batch::unsubscribe()
   {
     size_t prev = _subscriber_count.fetch_sub(1, std::memory_order_relaxed);
     if (prev == 0) {
       _subscriber_count.fetch_add(1, std::memory_order_relaxed);
       throw std::runtime_error("Cannot unsubscribe: subscriber count is already zero");
     }
   }

   size_t data_batch::get_subscriber_count() const
   {
     return _subscriber_count.load(std::memory_order_relaxed);
   }
   ```

   memory::memory_space* data_batch::get_memory_space() const

     return &_data->get_memory_space();
   }

   void data_batch::set_data(std::unique_ptr<idata_representation> data)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public API -> atomic ops | subscriber count operations use lock-free atomics |
| private accessors -> _data | null check in get_memory_space prevents null dereference |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | subscriber count underflow | mitigate | unsubscribe() checks prev==0, restores count, throws std::runtime_error. Prevents unbounded underflow wrapping. |
| T-02-02 | Information Disclosure | get_memory_space null deref | mitigate | Null check on _data before dereferencing: `if (_data == nullptr) { return nullptr; }`. Prevents crash on empty batch. |
| T-02-03 | Tampering | memory_order_relaxed subscriber count | accept | Subscriber count is informational (D-22). Relaxed ordering means count may be stale across threads but never corrupted. Acceptable for this use case per Open Question 1 in RESEARCH.md. |
</threat_model>
